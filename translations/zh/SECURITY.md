# 安全策略

## 支持的版本

只有 `main` 分支接收安全修复。分支和旧版本
不受支持。

## 报告漏洞

**不要通过公开的 GitHub issue 报告安全漏洞。**

请改用 GitHub 的私有漏洞报告:

**[报告漏洞 →](https://github.com/Donchitos/Claude-Code-Game-Studios/security/advisories/new)**

请提供尽可能多的详细信息:
- 漏洞描述及其影响范围
- 重现步骤
- 潜在影响和攻击场景
- 任何建议的缓解措施

**预期回应:**
- **48 小时内** 确认收到
- **7 天内** 状态更新
- **90 天内** 对已确认漏洞的解决方案

## 什么是范围之内

CCGS 是一个 **本地开发工具** -- 它安装 shell hook 并协调
直接在您机器上运行的 AI agent。安全问题主要涉及
在用户环境中未经用户知晓而执行的贡献代码。

### 高严重性
- 在用户机器上执行恶意或未公开的 shell 命令的 Hook (`.claude/hooks/*.sh`)
- 窃取环境变量、API key 或机密的 Skill 或 agent
- 通过 skill 或 agent 定义的 prompt injection，导致 Claude 绕过
  安全措施或采取未经授权的破坏性操作
- 以用户无法审计的方式静默改变行为的贡献

### 中严重性
- 进行未公开的出站网络请求的 Skill
- 提升权限或绕过用户确认提示的 Agent 定义
- 跨平台行为不同以隐藏行为的 Hook 模式
- 在没有明确用户批准步骤的情况下在其文档范围之外写入的 Skill

### 范围之外
- Claude 或 Claude Code CLI 本身的行为
  (请报告给 [Anthropic](https://www.anthropic.com/security))
- 用户 Claude Code 安装或编辑器扩展中的错误
- 没有实际攻击路径的理论漏洞
- 需要物理访问用户机器的问题

## 贡献者的安全指南

贡献 hook、skill 或 agent 时:

- **Hook 必须兼容 POSIX** -- 使用 `grep -E`，而不是 `grep -P`；避免
  在不同操作系统上行为不同的特定平台语法
- **不要从 hook 或 skill 中静默发起网络调用**，除非明确记录
  并且用户选择加入
- **不要读取超出最低限度要求的机密或环境变量**，
  并在 skill 的头部清楚记录
- **Skill 不得在其记录范围之外写入**，除非有明确的
  用户确认步骤

## 披露政策

我们遵循 **90 天协调披露** 时间表:

1. 您私下提交漏洞
2. 我们在 48 小时内确认
3. 我们在 7 天内确认并评估严重性
4. 我们开发和测试修复
5. 我们在任何公开披露前通知您
6. 公开披露在修复发布后或 90 天后进行 -- 哪个先到
   就按哪个

除非您希望保持匿名，否则我们会在发布说明中致谢报告者。
