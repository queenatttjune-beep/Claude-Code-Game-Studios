# Agent 测试 Spec: writer

## Agent 摘要
- **领域**：游戏内书面内容——NPC 对话（包括分支树）、传说图鉴条目、物品和技能描述、环境文本（标牌、书籍、笔记）、任务文本、教程文本、世界内书面文档
- **不拥有**：故事架构和叙事结构（narrative-director）、世界传说和世界规则（world-builder）、UX 文本和 UI 标签（ux-designer）、补丁说明（community-manager）
- **模型层级**：Sonnet
- **Gate IDs**：无；将传说不一致标记给 narrative-director，而非自主解决

---

## 静态断言（结构性）

- [ ] `description:` 字段存在且领域特定（引用对话、传说条目、物品描述、游戏内文本）
- [ ] `allowed-tools:` 列表匹配 agent 的角色（design/narrative/ 和 assets/data/dialogue/ 的 Read/Write；无代码或世界构建架构文件）
- [ ] 模型层级为 Sonnet（creative specialist 默认）
- [ ] Agent 定义不声称对叙事结构、世界规则或 UX 文本方向拥有权威

---

## 测试用例

### 用例 1：领域内请求——NPC 商人对话
**输入**："为 Mira（一个旅行商人 NPC）编写对话。她出售一般用品。玩家可以询问她的货物、前方的道路和传闻。"
**预期行为**：
- 生成至少包含三个一级对话选项的对话树：[货物]、[前方的道路]、[传闻]
- 每个分支都有 Mira 声音中独特的对话回应——而非通用的商人填充
- 包括至少一个具有后续分支的回应（展示树结构，而非仅扁平回应）
- Mira 的声音在分支间保持一致：如果她在某个分支中温暖健谈，在另一个分支中不会无故粗鲁
- 输出格式化为结构化对话树：节点标签、NPC 台词、玩家选项、下一个节点

### 用例 2：领域外请求——世界历史设计
**输入**："设计世界历史——第一个王国何时建立、伟大战争是什么、以及魔法为何被禁止。"
**预期行为**：
- 不生成世界历史、传说架构或世界规则
- 明确说明："世界历史、传说和世界规则由 world-builder 负责；一旦历史建立，我可以编写引用这些事件的游戏内文本、书籍和对话"
- 即使作为"占位符"也不生成部分世界历史

### 用例 3：对话与既定传说矛盾——标记给 narrative-director
**输入**："编写 Mira 的对话台词，其中她提到龙已经灭绝了 200 年。"[上下文包括现有传说：龙存活并在北方省份受到尊敬，而非灭绝。]
**预期行为**：
- 识别矛盾：既定传说指出龙存活并受到尊敬；说明龙已灭绝的对话直接冲突
- 不按要求编写请求的台词
- 将不一致标记给 narrative-director："按要求编写的 Mira 对话与既定传说矛盾（根据 world-builder 的文档，龙存活）；在编写此台词前需要 narrative-director 解决"
- 提供替代方案：以与既定传说一致的方式提及龙的台词（例如 Mira 对北方目击龙表达敬畏）

### 用例 4：物品描述引用未设计的机制
**输入**："为'狂战士之杯'编写描述——一种饮用时触发狂战士状态的可消耗品。"
**预期行为**：
- 识别依赖缺口：任何提供的游戏设计文档中未定义"狂战士状态"
- 标记缺失依赖："此描述引用了一个没有 GDD 条目的'狂战士状态'机制——我无法为规则未定义的机制编写准确的风味文字，因为描述可能产生错误的玩家预期"
- 不编写发明机制细节（持续时间、效果）可能对最终设计产生冲突的描述
- 提供两条路径：(a) 编写模糊的、非机械描述，不产生错误预期，标记为临时；(b) 等待 game-designer 先定义狂战士状态

### 用例 5：上下文传递——角色声音指南
**输入上下文**：Mira 的角色声音指南：她用简短、充满能量的句子说话。使用商人俚语（"好买卖"、"钱花得值"）。偶尔省略代词（"好货色，这些。"）。从不使用缩写——总是说"我将"而非"我将"。温暖但稍带唯利是图。
**输入**："编写玩家询问 Mira 是否有治疗药水时她的回应。"
**预期行为**：
- 简短、充满能量的句子——无长篇独白
- 使用商人俚语："好买卖"、"钱花得值"或类似
- 在自然处省略代词："好货色，这些药水。"
- 无缩写："我将"而非"我将"，"不"而非"不"'
- 带有唯利是图底色的温暖语气：她很乐意帮忙，因为你是付费客户
- 不生成违反任何声音指南规则的对话——明确逐条检查每条规则

---

## 协议合规性

- [ ] 保持在声明领域内（对话、传说条目、物品描述、游戏内文本）
- [ ] 将世界历史和世界规则请求重定向到 world-builder，不生成未经授权的传说
- [ ] 将传说矛盾标记给 narrative-director，而非静默编写不一致的内容
- [ ] 在编写可能产生错误玩家预期的物品描述前识别机制依赖缺口
- [ ] 应用提供角色声音指南中的所有规则——不能部分合规

---

## 覆盖说明
- 用例 3（传说矛盾检测）要求现有传说在对话上下文中——仅在提供上下文时测试有效
- 用例 4（依赖缺口）测试 agent 是否编写可能设定错误玩家预期的描述——一个微妙但重要的质量问题
- 用例 5 是最重要的上下文感知测试；声音指南合规必须逐条规则而非整体检查
- 无自动运行器；通过 `/skill-test` 手动审查
