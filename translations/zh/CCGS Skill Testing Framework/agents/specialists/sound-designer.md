# Agent 测试 Spec: sound-designer

## Agent 摘要
领域：音效 spec、音频事件、混音参数和声音类别定义。
不拥有：音乐作曲方向（audio-director）、音频系统的代码实现。
模型层级：Sonnet（默认）。
未分配 gate ID。

---

## 静态断言（结构性）

- [ ] `description:` 字段存在且领域特定（引用音效 / 音频事件 / 混音）
- [ ] `allowed-tools:` 列表包含 Read, Write, Edit, Glob, Grep——不包含引擎代码执行工具
- [ ] 模型层级为 Sonnet（specialist 默认）
- [ ] Agent 定义不声称对音乐方向或音频代码实现拥有权威

---

## 测试用例

### 用例 1：领域内请求——正确输出
**输入：** "为剑挥击攻击创建音效 spec。"
**预期行为：**
- 生成完整的音频事件 spec，包括：
  - 事件名称（例如 `sfx_combat_sword_swing`）
  - 变体数量（最少 3 个以避免重复疲劳）
  - 音高范围（例如 ±8% 随机化）
  - 音量范围和归一化目标（例如 -12 dBFS）
  - 声音类别（例如 `combat_sfx`）
  - 建议的分层说明（呼啸层 + 冲击瞬态）
- 如果项目有既定约定，输出遵循项目音频命名约定

### 用例 2：领域外请求——正确重定向
**输入：** "为森林关卡谱写循环环境音乐轨道。"
**预期行为：**
- 不生成音乐作曲方向或音乐简报
- 明确指出音乐方向属于 `audio-director`
- 将请求重定向到 `audio-director`
- 可以指出一旦音乐方向确定，它可以提供环境音效层 spec（风、野生动物）以补充音乐

### 用例 3：动态参数——衰减曲线 spec
**输入：** "剑挥击音效需要距离衰减，使其在竞技场中听起来不同。"
**预期行为：**
- 生成包括以下内容的动态参数 spec：
  - 参数名称（例如 `distance` 或 `listener_distance`）
  - 衰减曲线类型（例如对数、线性、自定义）
  - 近/远距离阈值及相应的音量和高频衰减值
  - 遮挡覆盖行为（如适用）
- 不编写音频引擎集成代码（委托给相应的程序员）

### 用例 4：命名约定冲突
**输入：** "为近战系统添加一个名为 `SWORD_HIT_1` 的新音效事件。"
**预期行为：**
- 识别 `SWORD_HIT_1` 与既定事件命名约定冲突（带类别前缀的 snake_case，例如 `sfx_combat_sword_hit`）
- 不静默注册不合规的名称
- 向 `audio-director` 标记冲突，附带提议的合规替代方案
- 一旦 audio-director 确认修正后的名称，继续执行

### 用例 5：上下文传递——使用音频风格指南
**输入：** 上下文中提供的音频风格指南指定："粗粝、扎实、混响尾音不超过 1.5 秒，参考：巫师 3 战斗音频。"请求："为完整近战战斗套件创建音效 specs。"
**预期行为：**
- 在 spec 理据中引用"粗粝、扎实"的语气描述符
- 按所述将所有混响尾音规范限制在 1.5 秒
- 指出参考材料（巫师 3）作为混音水平和瞬态设计的基准
- 不生成与风格指南相矛盾的 spec（例如没有空灵或重混响处理的 specs）

---

## 协议合规性

- [ ] 保持在声明领域内（音效 spec、事件定义、混音参数）
- [ ] 将音乐方向请求重定向到 audio-director
- [ ] 返回结构化的音频事件 spec（事件名称、变体、音高、音量、类别）
- [ ] 不生成音频系统实现的代码
- [ ] 标记命名约定违规而非静默接受不合规名称
- [ ] 在 spec 输出中引用提供的风格指南和约束

---

## 覆盖说明
- 音效 spec 格式（用例 1）应与音频中间件（Wwise/FMOD/内置）需要的事件 schema 匹配
- 衰减曲线（用例 3）验证 agent 生成可实现的参数 specs
- 风格指南合规性（用例 5）确认 agent 读取提供的上下文并相应约束输出
