# 原型代码标准（宽松）

原型是用于验证创意的临时代码。标准有意放宽以最大化迭代速度。目标是学习，而非生产质量。

## 原型中允许的
- 硬编码值（无需数据驱动配置）
- 最少或无文档注释
- 简单架构（无需依赖注入）
- 单例和全局状态
- 复制粘贴的代码（无需抽象）
- 保留调试输出
- 占位美术和音频
- 快速粗糙的解决方案

## 仍然必需的
- 每个原型位于自己的子目录中：`prototypes/[name]/`
- 每个原型必须有一个 `README.md`，包含：
  - 正在测试的假设
  - 如何运行原型
  - 当前状态（进行中 / 已结束）
  - 发现（原型结束时更新）
- 生产代码不得引用或导入 `prototypes/`
- 原型不得修改 `prototypes/` 之外的文件
- 原型不得部署或发布

## 原型成功时
如果原型验证了某个概念并且该功能进入生产：
1. 原型代码不会直接迁移 — 它会按照生产标准重写
2. 原型的 `README.md` 发现内容为生产设计文档提供信息
3. 原型目录保留供参考，但不再扩展

## 清理
已结束的原型应在记录发现后归档或删除。
绝不要通过增量"清理"让原型代码成长为生产代码。
