# 设计文档规则

- 每个设计文档必须包含以下 8 个章节：概述、玩家幻想、详细规则、公式、边界情况、依赖关系、调节旋钮、验收标准
- 公式必须包含变量定义、预期值范围和示例计算
- 边界情况必须明确说明会发生什么，而不仅仅"优雅处理"
- 依赖关系必须是双向的 — 如果系统 A 依赖 B，B 的文档必须提及 A
- 调节旋钮必须指定安全范围和它们影响的游戏属性
- 验收标准必须可测试 — QA 测试员必须能够验证通过/失败
- 不准确的说法："系统应该感觉良好"不是有效的规格说明
- 平衡数值必须链接到其来源公式或原理
- 设计文档必须逐步编写：先创建骨架，然后逐个章节填写，每个章节需经用户批准。每个批准的章节立即写入文件，以持久化决策并管理上下文
